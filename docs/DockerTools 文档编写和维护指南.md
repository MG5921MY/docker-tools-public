## ✍️ 文档编写规范

### 1. Markdown 格式规范

#### 标题层级

```markdown
# 一级标题（文档标题）- 每个文档只有一个

## 二级标题（主要章节）

### 三级标题（子章节）

#### 四级标题（详细说明）

##### 五级标题（尽量避免过深）
```

#### 代码块

````markdown
# Shell 命令
```bash
docker ps
```

# YAML 配置
```yaml
version: '3.8'
services:
  app:
    image: nginx
```

# 输出示例
```
[INFO] 导出成功
```
````

#### 强调和标记

```markdown
**粗体** - 重要概念
*斜体* - 术语或引用
`代码` - 命令、文件名、变量
⭐ - 重点推荐
⚠️ - 警告提示
✅ - 成功或推荐
❌ - 错误或不推荐
```

### 3. 内容编写规范

#### 中英文混排

```markdown
# ✅ 推荐
Docker Compose 是一个...

# ✅ 推荐（双语）
Docker Compose / Docker 编排工具

# ❌ 避免
DockerCompose是一个...（缺少空格）
```

#### 代码示例

```markdown
# ✅ 好的示例
```bash
# 导出单个容器
./docker-export-compose.sh my-container

# 查看帮助
./docker-export-compose.sh --help-cn
```

# ❌ 不好的示例
```bash
./docker-export-compose.sh my-container  # 没有注释说明
```
```

#### 表格使用

```markdown
# ✅ 清晰的表格
| 功能 | 说明 | 版本 |
|------|------|------|
| 导出 | 单容器导出 | v1.0+ |
| 批量 | 批量导出 | v2.0+ |

# ✅ 对比表格
| 特性 | v2.1 | v2.2 |
|------|------|------|
| 隐私模式 | ❌ | ✅ |
```

## 🎯 文档类型指南

### 类型 1: 教程类文档

**目的**: 教学、入门  
**示例**: Docker 基础教学.md

**结构要求**:
- 循序渐进，由浅入深
- 大量示例和截图
- 实战练习
- 常见问题解答

**编写要点**:
```markdown
1. 明确受众（新手/有经验）
2. 提供完整代码示例
3. 解释每个步骤
4. 包含预期输出
5. 提供参考答案
```

---

### 类型 2: 参考类文档

**目的**: 查询、速查  
**示例**: QUICK_REFERENCE.md

**结构要求**:
- 简洁明了
- 表格化信息
- 快速定位
- 最小化解释

**编写要点**:
```markdown
1. 使用表格组织信息
2. 按类别分组
3. 提供命令模板
4. 包含常见场景
```

---

### 类型 3: 指南类文档

**目的**: 详细说明、最佳实践  
**示例**: SECURITY_GUIDE.md

**结构要求**:
- 详细全面
- 场景化
- 包含警告
- 提供检查清单

**编写要点**:
```markdown
1. 明确警告和建议
2. 提供完整场景示例
3. 包含安全检查清单
4. 故障排查指导
```

---

### 类型 4: 变更类文档

**目的**: 记录更新、版本对比  
**示例**: CHANGELOG_v2.2.md

**结构要求**:
- 按版本组织
- 分类清晰
- 向后兼容说明
- 迁移指南

**编写要点**:
```markdown
1. 新增功能详细说明
2. 破坏性变更警告
3. 版本对比表格
4. 升级步骤清晰
```

## 🔄 文档维护流程

### 1. 新增功能时

```markdown
步骤 1: 更新 CHANGELOG
- 在 CHANGELOG 中记录新功能
- 包含示例代码

步骤 2: 更新相关文档
- README.md（功能列表）
- QUICK_REFERENCE.md（命令速查）
- SECURITY_GUIDE.md（如涉及安全）

步骤 3: 更新版本号
- 所有相关文档的版本号
- 最后更新日期

步骤 4: 验证链接
- 确保所有文档链接有效
- 检查交叉引用
```

### 2. 修复问题时

```markdown
步骤 1: 更新 CHANGELOG
- 在 "已修复问题" 章节记录

步骤 2: 更新故障排查
- SECURITY_GUIDE.md 故障排查章节
- 添加解决方案

步骤 3: 更新常见问题
- Docker 基础教学.md 常见问题章节
```

### 3. 定期维护

```markdown
每月检查:
- [ ] 所有链接是否有效
- [ ] 代码示例是否可运行
- [ ] 版本号是否一致
- [ ] 日期是否更新

每季度检查:
- [ ] 文档结构是否合理
- [ ] 是否有过时内容
- [ ] 用户反馈收集
- [ ] 优化改进
```

## 🌍 国际化考虑

### 中英文双语

```markdown
# 重要概念使用双语
Docker Compose / Docker 编排工具

# 命令和输出保持英文
```bash
docker compose up -d
```

# 注释可以双语
```yaml
# 数据卷配置 / Volume configuration
volumes:
  - ./data:/app/data
```
```

### 术语统一

| 英文 | 中文 | 说明 |
|------|------|------|
| Container | 容器 | 始终使用"容器"而非"集装箱" |
| Image | 镜像 | 始终使用"镜像" |
| Compose | Compose | 保持英文，或"编排工具" |
| Volume | 数据卷 | 或"卷" |
| Network | 网络 | - |
