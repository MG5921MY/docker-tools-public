## 🎯 文档组织原则

### 1. 层级分离

```
第 1 层：主目录
└─ 项目级别的总览和通用文档

第 2 层：docs/ (项目文档)
└─ Docker 基础教学、文档规范等

第 3 层：export/sh/ (脚本目录)
└─ 脚本本身和基础 README

第 4 层：export/sh/docs/ (脚本详细文档)
└─ 脚本的详细文档，按编号组织
```

### 2. 编号规范

```
0_README.md       # 索引文件，总是第一个
1_*.md            # 最重要/必读的文档
2_*.md            # 次重要的文档
3_*.md            # 常用参考
4_*.md            # 详细说明
5_*.md            # 历史记录
```

### 3. 文件命名

**✅ 推荐**:
- 使用数字编号前缀
- 使用中文描述性名称
- 使用下划线分隔

**示例**:
- `0_README.md`
- `1_安全使用指南.md`
- `2_配置文件指南.md`

## 🔧 如何添加新文档

### 步骤

```bash
1. 确定文档类型和重要性
   └─ 决定编号（1-5）

2. 创建文档
   └─ 使用 UTF-8 编码
   └─ 使用编号_名称.md 格式

3. 更新索引
   └─ 更新 export/sh/docs/0_README.md
   └─ 更新 export/sh/README.md（如需要）

4. 更新链接
   └─ 确保相关文档的链接正确
```

### 示例

```bash
# 如果要添加性能优化指南
cd export/sh/docs

# 创建文档（编号 6）
cat > 6_性能优化.md <<'EOF'
# 性能优化指南
...
EOF

# 更新 0_README.md 索引
# 添加新文档到列表
```

## 📝 维护指南

### 保持目录整洁

1. **新增脚本** → 放在 `export/` 目录对应子目录
2. **新增文档** → 放在 `docs/` 目录并编号
3. **配置文件** → 放在脚本同级目录
4. **避免在根目录** → 堆积过多文件

### 文档更新

1. **功能更新** → 更新对应编号的文档
2. **版本发布** → 更新 5_更新日志.md
3. **新增功能** → 可能需要新建文档
